.env explanation

If you develop or deploy go4lage you need to adjust the .env file. As shown in the setup there are two .env files prepared. one for native and one for docker. 

# .env
DB_URL=postgres://go4lage:go4lage@localhost:5432/go4lage?sslmode=disable
GOOSE_DBSTRING="user=go4lage password=go4lage dbname=go4lage sslmode=disable"
GOOSE_DRIVER=postgres 
BASEURL=http://127.0.0.1
APIURL=http://localhost
PORT=8080
USER_TOKEN_VALID_MINS=2400
SUPERUSER_2FA=false
SUPERUSER_TOKEN_VALID_MINS=600
LOGINTHROTTLE_TIME_S=1
DEBUG=true

DB_URL=postgres://go4lage:go4lage@db:5432/go4lage?sslmode=disable

This is the information for the postgres driver. username, password, database,host,port.

GOOSE_DBSTRING="host=db user=go4lage password=go4lage dbname=go4lage sslmode=disable"

This is the information for the built in goose driver.

GOOSE_DRIVER=postgres

This is the information for the built in goose driver.

BASEURL=http://127.0.0.1

your base url. Change this to https://example.com

APIURL=http://localhost

your apiurl. Change this to your apiurl (needed for more complex setups)

PORT=8080

The Port of this App. Make this consistent with the Docker build.

USER_TOKEN_VALID_MINS=2400

Specifies how long a bearer Token for a user is valid. After this time the user has to log in again.

SUPERUSER_2FA=true

If this is true, then 2fa for superusers is mandatory. If you activate this later, then you have to create new superusers.

SUPERUSER_TOKEN_VALID_MINS=600

This is the same as for normal users. But it could be needed to have a much shorter time here for security reasons.

LOGINTHROTTLE_TIME_S=1

This is the auth throttle time. One ip adress has to wait this value in seconds before it can try to log in again, after they put in wrong credentials.

DEBUG=true

Not fully implemented.
