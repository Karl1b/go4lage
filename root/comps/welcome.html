Go4lage

Go4Lage serves as the ideal platform for launching small to medium-sized projects, whether you need a backend for your application or a server-side rendered web application.

Crafted in Go, it offers simplicity and exceptional speed. Designed for self-hosting within a Docker Compose network, Go4Lage ensures significant cost savings for your company, making it more competitive for small to medium-sized enterprises. It includes a comprehensive range of features.

Its goal is to provide 80% of all batteries included at 20% of the expense.

Features of Go4lage

  • Straightforward and simple code.
  • Easy deployment, suitable even for beginners.
  • Group and permission system
  • Backups
  • Logging
  • Integrated caching (no need for Redis).
  • Integrated cache buster for static files.
  • Optional 2FA for superusers.
  • Documentation and an example admin dashboard included.
  • Clear sepparation of build-, init- and runtime
  • Distinct separation of build, initialization, and runtime phases.

FAQ

Question Explanation
What does Go4lage mean? Go4lage is a play on words. 'Vorlage' (4lage) is German for template, so it could roughly be interpreted as 'GoTemplate.'
Will there ever be Redis support? Redis support is not planned as Go offers a more native caching solution. Alternatively, you can use Postgres in memory mode for SQL features.
Is Go4lage really free? Yes, Go4lage is free both as in free speech and as in free beer, even for commercial use.
How can I contribute? Please refer to the official GitHub repository for details.
Why is 2FA available for superusers but not for normal users? 2FA requires a well-thought-out setup. It involves more than just providing a key secret to users. To secure your app effectively, proper planning is necessary. If you consider this, you will be able to configure Go4lage accordingly.
I come from Django (Python), is Go hard to learn? Not at all. Only the initial steps might seem challenging. Golang values simplicity and is often more straightforward than Python because it avoids unexpected complexities.
Why is there no ORM? ORMs introduce an unnecessary layer of complexity. We believe in starting with the data you want to manipulate, as the most important aspect of information technology is the data stored in the database.
How powerful (fast) is Go4lage? Although there are no real-world tests published yet, Go applications are generally fast enough. The main appeal of Go is not just its speed but its simplicity and the ability to write concurrent code, which aids in rapid development and producing robust code that can adapt to future requirements.
